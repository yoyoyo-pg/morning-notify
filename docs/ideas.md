## イベント・地域情報通知

> **備考**: このボットは「**厚切りジェイソン**」として実装済み。朝夕通知の「ショーンK」と並ぶ存在として、Discordの `#events` チャンネルを担当する想定。

### 概要

名古屋のイベント情報を定期的にDiscordへ通知する。毎日の朝通知とは別チャンネルで、週2回程度の配信を想定。

### やりたいこと

- **久屋大通のイベント情報** — 公園・広場でのイベントスケジュール
- **名古屋市内のイベント全般** — 週末のイベント、フェス、マルシェ等
- その他、定期的に知りたいローカル情報

### 実装イメージ

```
# 現在（毎朝6時、#morning チャンネル）
☀️ おはようございます...

# 追加したいもの（週2回、#events チャンネル）
🎪 今週末の名古屋イベント情報
	・〇〇マルシェ（久屋大通公園 / 5/3〜5/4）
	・〇〇フェスティバル（鶴舞公園 / 5/4）
	...
```

### 技術的な検討事項

**情報ソース候補**
| ソース | 特徴 | 取得方法 |
|--------|------|----------|
| 名古屋市観光情報サイト | 公式情報 | スクレイピング or RSS |
| 久屋大通公園公式サイト | ピンポイントで欲しい情報 | スクレイピング |
| じゃらんnet・イベントチェッカー | イベント情報が豊富 | RSS or スクレイピング |
| Peatix | 有料・無料イベント | RSS あり |

**スケジュール**
- 現在のcron（毎朝6時）とは別のワークフローを追加
- 例: 月・木の朝6時に実行（`0 21 * * 1,4` UTC）

**Discord チャンネル構成（案）**
| チャンネル | 内容 | 頻度 |
|-----------|------|------|
| `#morning` | 天気・カレンダー・ニュース | 毎日 |
| `#events` | 名古屋イベント情報 | 週2回 |

→ `DISCORD_WEBHOOK_URL_EVENTS` として別のWebhook URLをSecretsに追加する想定

### 実装の進め方（案）

1. 情報ソースを決める（どこから取るか）
2. `src/events.py` を新規作成
3. `src/main_events.py`（イベント通知専用のエントリーポイント）を作成
4. `.github/workflows/events-notify.yml` を追加
5. Discord に `#events` チャンネルと専用Webhookを用意

### 懸念点

- スクレイピングはサイト構造の変更で壊れやすい
- 公式RSSがあるかどうかが鍵
- イベント情報の鮮度管理（過去のイベントを通知しないようにする）

### 🔥 松岡修造（タスクリマインダー）

**役割**: 昼にNotionの未完了タスクを激励つきで通知する  
**タイミング**: 毎日 12:00  
**データソース**: Notion API（当日ジャーナルの未完了チェックボックス）

```
🔥 まだやれる！絶対できる！諦めるな！

今日やるって決めたこと、まだ 3個 残ってるぞ！
太陽より熱くなれ！！！

📋 未完了タスク
  ☐ 企画書レビュー
  ☐ テストコード書く
  ☐ 請求書送る

日本一熱い男より愛を込めて 🔥🔥🔥
```

**実装メモ**
- `matsuoka/` ディレクトリで管理（ショーンK・厚切りジェイソンと同様に分離）
- Notion API で当日ジャーナルの未完了タスクを取得
- タスク0件のときは「全部やり切ったな！俺は感動した！！」的なメッセージを出す

### 😱 出川哲朗（エラー・障害アラート）

**役割**: GitHub Actions の失敗時にだけ発火するアラートボット  
**タイミング**: ワークフロー失敗時のみ（定期実行ではない）  
**データソース**: GitHub Actions のワークフロー失敗イベント

```
😱 やばいよやばいよやばいよ！！

morning-notify.yml が失敗しちゃってるよ！！
ショーンKからの朝の通知が届いてないかも！！

❌ エラー内容
  DISCORD_WEBHOOK_URL が設定されてないっぽいよ！！

早く直してよ〜〜頼むよ〜〜！！ 😱😱😱
```

**実装メモ**
- 既存ワークフローに `on: workflow_run` トリガーを追加する形で実装
- 専用ディレクトリではなく `.github/workflows/alert-notify.yml` 単体で完結できるかも

### 💄 マツコ・デラックス（週次レビュー）

**役割**: 金曜夜に1週間を辛口でまとめる週次振り返り通知  
**タイミング**: 金曜 21:00（ショーンKの夜通知の代わりに週1だけ発火）  
**データソース**: Notion API（今週のジャーナルの完了・未完了タスク集計）

```
💄 あら、今週も終わりね。

📊 今週（4/28〜5/2）の振り返り

完了: 8件 ／ 未完了: 3件

未完了のやつ、来週こそやんなさいよ。
あなたにはできるって私は思ってるけど、
本当にやる気あるのかしらね。

まあ、お疲れ様。ゆっくり休みなさい。💄
```

**実装メモ**
- `matsuko/` ディレクトリで管理
- 金曜のみ実行するcron（`0 12 * * 5` UTC = 金曜21時JST）
- Notion APIで今週分のジャーナルページを集計（タスクの完了率を出す）
- Notion未連携の場合は固定メッセージのみ

### 💪 ジョージ（朝の男前通知）【迷い中】

**役割**: 朝のショーンKの代替候補。静かな圧と断定で一日を始める  
**タイミング**: 毎日 6:00（ショーンKと役割を入れ替える形）  
**データソース**: ショーンKと同様（天気・スケジュール・ニュース）

```
💪 起きろ！5/1(木)だ！

男は言い訳をしない。
今日やるべきことをやれ。それだけだ。

📅 本日のスケジュール
  ・10:00 週次MTG
  ・14:00 1on1

お前が動かなければ、誰も動かない。
今すぐ始めろ。💪
```

**実装メモ**
- `src/reminkun/` のコードはそのまま流用し、文言だけ変える
- 松岡修造との違い：修造は「熱量で押す」、ジョージは「断定と静圧」
- ショーンKを夜専任にする場合、朝担当としてジョージを立てる選択肢がある

## AI 運用・ワークフロー改善メモ

このプロジェクトを通じて得た AI との協働ノウハウ。今後の開発運用に活かす。

### ワークスペースを用途ごとに分ける

1つの巨大なプロジェクトに全部入れるのではなく、用途ごとに小さく分ける。

| ワークスペース | 用途 |
|-------------|------|
| スキル・自動化用 | スクリプト、ワークフロー |
| 知識管理用 | メモ、調査結果 |
| 資産管理用 | 財務・所有物の記録 |
| 日々のタスク管理用 | TODO、進捗 |

共通点：Markdown で情報を残し、Claude Code から読ませたり更新したりする。

### 新しいワークスペースを作ったときの初動

```
/init
「日本語で使いたい」
「ここで取れる最低限のセキュリティを設定したい」
「このプロジェクトでは何に気を付けるべきか教えて」
```
