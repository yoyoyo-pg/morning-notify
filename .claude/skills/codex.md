# Codex

Codex CLIを使用してコードレビュー・分析を実行するスキル。

## 実行コマンド

```bash
codex exec --full-auto --sandbox <sandbox> --cd <project_directory> "<request>"
```

モデルはデフォルト（`gpt-5.5`）のまま使用する。別のモデルを使いたい場合は `-c model=<model>` で指定できる。

## sandbox の使い分け

| 用途 | sandbox オプション | 説明 |
|------|-------------------|------|
| レビュー・分析・調査 | `--sandbox read-only` | ファイルを読むだけ（安全） |
| コードの修正・生成 | `--sandbox workspace-write` | ファイルへの書き込みを許可 |

修正を依頼する場合は `workspace-write` を使うこと。`read-only` のままだと Codex はファイルを変更できず、提案だけ返してくる。

## プロンプトのルール

Codex に渡すリクエストには以下を必ず含めること：

1. **簡潔さの指示** — 出力が長くなりすぎると末尾が切れるため
   > 「回答は要点を箇条書きで簡潔にまとめてください。」

2. **自律実行の指示** — 確認待ちで止まらせないため
   > 「確認や質問は不要です。具体的な提案・修正案・コード例まで自主的に出力してください。」

3. **必要に応じてコンテキストを埋め込む** — Claude が持つ情報（エラー内容、読んだファイルの要点など）はプロンプトの冒頭に直接書くこと

## エラー時の対応

| エラー | 原因 | 対応 |
|--------|------|------|
| `Quota exceeded` | APIクレジット不足 | ユーザーに OpenAI の課金状況確認を依頼 |
| `Reconnecting...` 後に失敗 | 一時的な接続エラー | 1回だけ再実行。それでも失敗すればユーザーに報告 |

## 使用例

### コードレビュー（読み取り専用）
```bash
codex exec --full-auto --sandbox read-only --cd /path/to/project \
  "このプロジェクトのコードをレビューして改善点を指摘してください。回答は要点を箇条書きで簡潔にまとめてください。確認や質問は不要です。具体的な修正案まで自主的に出力してください。"
```

### バグ調査（コンテキスト付き）
```bash
codex exec --full-auto --sandbox read-only --cd /path/to/project \
  "以下のエラーが発生しています。\n---\n<エラー内容をここに貼る>\n---\n原因を調査してください。回答は要点を箇条書きで簡潔にまとめてください。確認や質問は不要です。原因の特定と修正案まで自主的に出力してください。"
```

### コード修正（書き込み許可）
```bash
codex exec --full-auto --sandbox workspace-write --cd /path/to/project \
  "weather.py のエラーハンドリングを改善してください。回答は要点を箇条書きで簡潔にまとめてください。確認や質問は不要です。修正を直接ファイルに反映してください。"
```

### 設計相談
```bash
codex exec --full-auto --sandbox read-only --cd /path/to/project \
  "このプロジェクトのアーキテクチャを分析し、改善提案をしてください。回答は要点を箇条書きで簡潔にまとめてください。確認や質問は不要です。具体的な提案まで自主的に出力してください。"
```

## 実行手順

1. ユーザーの依頼が「レビュー・分析」か「修正・生成」かで sandbox を決める
2. 対象プロジェクトのディレクトリを特定する
3. 必要ならエラーや関連情報をプロンプトの冒頭に埋め込む
4. **簡潔さの指示と自律実行の指示を末尾に追加する**
5. `-c model=o4-mini` を付けてコマンドを実行
6. エラーが出たら上記のエラー表を参照して対応する
7. 結果をユーザーに報告する
