# ドキュメント管理

## 改善サイクル

1. **作業開始前**: `docs/lessons.md` を読み、関連する過去の教訓を確認する
2. **実装完了後（PR作成前）**: `docs/lessons.md` に教訓を追記し、PRに含める

## lessons.md の書き方

- **何をしたか** ではなく **なぜそうしたか・次回どうすべきか** を書く
- コードや git log で分かることは書かない
- 項目ごとに日付を付ける（例: `## 2026-05-01 ディレクトリ構成見直し`）
- ユーザーから指示されなくても、非自明な判断・ハマりどころがあれば都度追記する

## その他のドキュメント

| ファイル | 用途 |
|---------|------|
| `docs/lessons.md` | ハマりどころ・判断理由・次への教訓（蓄積型） |
| `docs/ideas.md` | 開発アイデアメモ |
| `docs/ai-coding-experience.md` | AIコーディング体験記 |
